# 1. Слоистая архитектура с портами и адаптерами
2026-05-14
Status: acepted
Это утилита, которая декодирует изображение, затем может его конверировать, обрезать и выполнять другие преображения, записывая изменённый вариант обратно на диск. Это моя дипломная работа потому что...

Мы принимаем слоистую архитектуру в духе Гексагональной архитектуры (Ports & Adapters):
domain (internal/domain/) — чистые типы и правила (форматы изображений, качество, уровни сжатия). Без I/O, без внешних зависимостей.
application (internal/app/) — сценарии использования (use cases), которые оркеструют домен, выраженные в терминах портов (интерфейсов), определённых в internal/app/ports/.
adapters (internal/adapter/) — конкретные реализации портов: кодеки на основе image/jpeg, image/png, golang.org/x/image/webp; слой CLI на основе spf13/cobra.
Зависимости направлены внутрь: adapter → app → domain. Домен никогда не импортирует ничего из app или adapter.
Публичные точки входа находятся в cmd/imgconv/. Всё остальное расположено в internal/, чтобы предотвратить импорт извне.
